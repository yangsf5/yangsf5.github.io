无计可施时试试神奇的重启

By 三疯

发表于 2018-01-08

文章目录
  1. 1. 重启 Chrome 浏览器解决空白页
  2. 2. 重启 Mac 日历解决同步不稳定

“重启”这种操作，在 Windows 时代，被教育成了常规解决方案，因为这个系统里的各种环境、依赖，实在错综复杂。类 Linux 系统下（包括 Mac 系统），这些关系比较容易理顺，所以大部分情况下都能找出问题原因。

知其所以然，也是知识追求者的基本功。但有时在有限时间内实在无计可施时（即使设计优良的类 Linux 系统），可以试试一些偏方，事后再去钻研真理。

重启 Chrome 浏览器解决空白页

我在 Mac 下默认浏览器是 Chrome。所以正常情况下，在企业 QQ 或者微信里里点击链接时，会自动在当前已经打开的 Chrome 窗口里新建一个标签页，然后在这个标签的地址栏里自动填上那个链接网址，然后自动访问到那个网址。

但最近这个自动操作失效了，每次在企业 QQ 或者微信点击链接时，会自动新开一个 Chrome 窗口，然后在新窗口里开的是一个空白标签页。没有自动填网址，也没有自动访问。

重启 Chrome。在 Dock 里点击 Chrome 的“退出”菜单，这样是整个 Chrome 退出，包括所有窗口。重新打开 Chrome，然后就 OK 了。

这一两天，又出现了。重启 Chrome 不管用了，然后再试试重启企业 QQ 或者微信，都不管用了。不想先尝试重启 Mac，就去搜了下。V2EX 上了解到，好多人都发现了，都集中在 Mac 10.13 系统上，大家猜测是这个版本的锅。大家也都是重启来解决的，各种组合的重启方式。也有说是 Chrome 自己的锅，每次升级 Chrome 都要重启，但我现在 Chrome 是没有升级的。

最后看到有网友回复的让人眼前一亮的方案：

在 Chrome 的地址栏输入：chrome://restart
浏览器会自动关闭并自动重启再自动打开你重启前的窗口。然后问题就解决了。

试了，果然奇效。这个方案比以前手动重启方便多了，多了自动打开并自动重开之前的窗口。

对于这个方式的奇效的原因，我猜测，可能是命令里做了些资源、环境自动清理的操作。希望这个方式一直奏效，至少多撑一段时间。

重启 Mac 日历解决同步不稳定

最近在 iPhone 上新建日历，不能自动更新到 Mac 日历上。手动点击同步菜单也不行。在 iCloud 里关掉日历的同步，而后再打开，就把历史全同步过来了。

但是再在 iPhone 里新建日历，Mac 上 日历又同步不了了。愁。

某天晚上联想到 Chrome 的效果，试试重启 Mac 的日历客户端呢？重启了，哇，果然就同步过来了。
