我的技术博客初衷

2016-07-03 更新

后来 2014 年将博客迁移到 WordPress，只是 WordPress 里只写新的，没有将这些老博客转过来。

最近将博客再次迁移到 Github Pages，因为这个像写代码似的，可定制性强，并且用 Git 做版本管理，以后如果再调整也方便了。而且博客的发表日期还可以写成很久前的，所以就想将以前的 C++ 博客园的博客也搬过来，感受自己的「积跬步」。

2013-04-08 更新

ChinaUnix 的博客不稳定，又迁回 C++ 博客园了。

2012-11-19 原博文

博客前前后后换了好几个，各个的初衷也不一。

大二时有个为了学英语，坚持写英语短篇而建的。可是英语很挫，没坚持多久。

再后来大概是大三了，转到百度空间了，就是写写自己都在做什么，在学习什么技术，心得，一些设计想法，一些问题搜索整理。

大四了，开始实习了，看了些前辈的博客，都在 ChinaUnix 和博客园等程序员博客专业些的地方。考虑下就搬到 C++ 博客园了，当时就冲着 C++ 这个明确的名字去的。大四一年挺忙的，又是全职实习，又是毕业设计，博客也没怎么写。很多想写的，只是占了个位置，“待补充……”待到现在也没补充。

毕业后进的搜狐畅游，搞游戏的，对代码管理比较严，工作机是没有外网的。工作也挺忙，一般都加班很晚，晚上回家就顾不上写什么博客了。

在畅游呆了两年多的时候，开始山地骑行了。很新鲜的事情嘛，就想写些游记、攻略总结什么的。这样的文章很私人，就最终决定在人人网上写了。还不错，去的经典的地方都写了。

本来准备也在人人上写技术日志时，发现大家不感冒，想想也是，人人上的好友大部分是以前同学，他们又不是搞 IT 的。

所以就先在麦库上写着，当自己的笔记算了。也考查过自己买 VPS 自己写主页，但是发现没那份精力。

五月份换了现在的工作，又有外网了，感觉像是“被放出来了”。工作电脑就有外网的直接好处是，工作中的问题能立刻去搜索资料，加快解决进度。很多棘手的问题，我习惯的将要点记到麦库上，方便下次再出现这些问题，这类问题，能很快温习上次的解决结果和解决手段。

网上的垃圾信息很多，有些问题，自己解决了，把过滤后的信息帖出来，方便别人，也督促自己总结。这就是我的技术博客的初衷。

技术博客选在 ChinaUnix，主要是对 ChinaUnix 有些感情。但是这里的博客审核有点令人蛋疼啊。

至于生活日志，还是记在人人上，我个人不喜欢把自己生活上的事情暴露太多在外边，被围观的感觉怪怪的，没人鸟的感觉更怪，哈哈。
